# 智慧校园平台需求文档

## 1. 项目概述

### 1.1 项目名称
智慧校园管理平台

### 1.2 项目背景
随着信息化技术的发展，传统的校园管理模式已经无法满足现代教育的需求。本项目旨在开发一个集教务管理、信息发布、在线学习于一体的智慧校园平台，提高校园管理效率，优化师生体验。

### 1.3 项目目标
- 实现教务管理的数字化和自动化
- 提供便捷的信息发布和查询渠道
- 搭建在线学习和作业管理平台
- 提升校园管理效率和服务质量

### 1.4 技术架构
- **前端**: Vue 3 + Element Plus / Ant Design Vue
- **后端**: Spring Boot + MyBatis Plus
- **数据库**: MySQL 8.0
- **其他**: Redis（缓存）、JWT（认证）

## 2. 用户角色定义

### 2.1 学生
- 查看个人课表和成绩
- 进行选课操作
- 查看通知公告和新闻
- 访问课程资源
- 提交作业

### 2.2 教师
- 管理所授课程
- 录入和管理学生成绩
- 发布课程资源
- 布置和批改作业
- 查看课表安排

### 2.3 管理员
- 管理用户账号（学生、教师）
- 管理课程信息
- 发布通知公告和新闻
- 管理选课规则
- 系统数据统计和维护

## 3. 功能需求

### 3.1 用户管理模块

#### 3.1.1 登录注册
- 用户登录（学号/工号 + 密码）
- 记住登录状态
- 密码找回功能
- 首次登录强制修改密码

#### 3.1.2 个人信息管理
- 查看个人基本信息
- 修改个人资料（头像、联系方式等）
- 修改登录密码

#### 3.1.3 用户管理（管理员）
- 学生信息管理（增删改查）
- 教师信息管理（增删改查）
- 批量导入用户信息
- 用户状态管理（启用/禁用）

### 3.2 教务管理模块

#### 3.2.1 课程管理（管理员）
- 课程信息管理（课程名称、学分、学时、课程简介）
- 开课计划管理（学期、上课时间、地点、任课教师）
- 课程容量设置
- 课程状态管理（开放选课/关闭选课）

#### 3.2.2 选课管理
**学生端：**
- 查看可选课程列表（支持筛选和搜索）
- 查看课程详细信息（教师、时间、地点、已选人数）
- 在线选课（检测时间冲突、容量限制）
- 退课操作（在规定时间内）
- 查看已选课程列表

**管理员端：**
- 设置选课时间段
- 查看选课统计数据
- 手动调整学生选课（特殊情况）

#### 3.2.3 成绩管理
**教师端：**
- 查看所授课程的学生名单
- 录入学生成绩（平时成绩、期末成绩）
- 修改成绩（需审批或限制次数）
- 导出成绩单

**学生端：**
- 查看个人成绩（按学期查询）
- 查看成绩统计（GPA、学分统计）
- 导出成绩单

**管理员端：**
- 查看全校成绩统计
- 成绩审核管理
- 成绩异常监控

#### 3.2.4 课表管理
**学生端：**
- 查看个人课表（周视图、日视图）
- 按学期切换课表
- 导出课表

**教师端：**
- 查看个人授课课表
- 查看教室使用情况

**管理员端：**
- 排课管理（自动排课/手动调整）
- 教室资源管理
- 课表冲突检测

### 3.3 信息发布模块

#### 3.3.1 通知公告管理
**管理员端：**
- 发布通知公告（标题、内容、附件、发布范围）
- 编辑和删除通知
- 设置通知优先级（普通/重要/紧急）
- 定时发布功能
- 查看通知阅读统计

**学生/教师端：**
- 查看通知列表（按时间排序）
- 查看通知详情
- 标记已读/未读
- 通知搜索功能
- 重要通知置顶显示

#### 3.3.2 新闻管理
**管理员端：**
- 发布校园新闻（标题、封面图、内容、分类）
- 编辑和删除新闻
- 新闻分类管理
- 新闻审核（可选）

**学生/教师端：**
- 浏览新闻列表（支持分类筛选）
- 查看新闻详情
- 新闻搜索
- 热门新闻推荐

### 3.4 在线学习模块

#### 3.4.1 课程资源管理
**教师端：**
- 上传课程资源（课件、视频、文档等）
- 资源分类管理（按章节组织）
- 编辑和删除资源
- 设置资源访问权限
- 查看资源下载统计

**学生端：**
- 浏览已选课程的资源列表
- 在线预览资源（支持PDF、视频等）
- 下载课程资源
- 资源搜索功能
- 收藏常用资源

#### 3.4.2 作业管理
**教师端：**
- 发布作业（标题、内容、截止时间、附件）
- 设置作业提交方式（在线提交/附件上传）
- 查看作业提交情况
- 批改作业（评分、评语）
- 作业统计分析（提交率、成绩分布）

**学生端：**
- 查看作业列表（待提交/已提交/已批改）
- 查看作业详情和要求
- 在线提交作业
- 上传作业附件
- 查看作业成绩和教师评语
- 作业提醒（临近截止时间）

## 4. 非功能性需求

### 4.1 性能要求
- 系统响应时间：普通操作 < 2秒，复杂查询 < 5秒
- 支持并发用户数：至少500人同时在线
- 数据库查询优化，关键表建立索引
- 使用Redis缓存热点数据

### 4.2 安全性要求
- 用户密码加密存储（BCrypt）
- 使用JWT进行身份认证
- 接口权限控制（基于角色的访问控制）
- 防止SQL注入、XSS攻击
- 敏感操作日志记录
- 文件上传类型和大小限制

### 4.3 可用性要求
- 界面简洁友好，符合用户使用习惯
- 提供操作提示和错误信息
- 支持主流浏览器（Chrome、Firefox、Edge）
- 响应式设计，适配不同屏幕尺寸

### 4.4 可维护性要求
- 代码结构清晰，遵循开发规范
- 关键功能添加注释
- 前后端分离架构
- 统一的异常处理机制
- 日志记录系统

## 5. 数据库设计概要

### 5.1 核心数据表

#### 用户相关
- **user（用户表）**: 用户ID、用户名、密码、姓名、角色、状态、创建时间等
- **student（学生表）**: 学生ID、学号、班级、专业、年级、联系方式等
- **teacher（教师表）**: 教师ID、工号、职称、院系、联系方式等

#### 教务相关
- **course（课程表）**: 课程ID、课程名称、学分、学时、课程简介等
- **course_schedule（开课计划表）**: 计划ID、课程ID、教师ID、学期、上课时间、地点、容量等
- **course_selection（选课表）**: 选课ID、学生ID、课程计划ID、选课时间、状态等
- **grade（成绩表）**: 成绩ID、学生ID、课程ID、平时成绩、期末成绩、总成绩等

#### 信息发布相关
- **notice（通知公告表）**: 通知ID、标题、内容、发布人、发布时间、优先级等
- **news（新闻表）**: 新闻ID、标题、封面图、内容、分类、发布时间等
- **notice_read（通知阅读记录表）**: 记录ID、通知ID、用户ID、阅读时间等

#### 在线学习相关
- **course_resource（课程资源表）**: 资源ID、课程ID、资源名称、资源类型、文件路径、上传时间等
- **homework（作业表）**: 作业ID、课程ID、标题、内容、截止时间、创建时间等
- **homework_submit（作业提交表）**: 提交ID、作业ID、学生ID、提交内容、附件路径、提交时间等
- **homework_grade（作业成绩表）**: 成绩ID、提交ID、分数、评语、批改时间等

### 5.2 数据库设计原则
- 遵循第三范式，减少数据冗余
- 合理使用外键约束保证数据完整性
- 为常用查询字段建立索引
- 使用逻辑删除而非物理删除重要数据
- 时间字段统一使用DATETIME类型

### 6.1 整体风格
- 采用现代化扁平设计风格
- 主色调建议：蓝色系（体现科技感和专业性）
- 界面布局清晰，信息层次分明
- 统一的UI组件库（Element Plus 或 Ant Design Vue）

### 6.2 主要页面
**前台页面：**
- 登录页面
- 首页（展示通知、新闻、快捷入口）
- 个人中心
- 课表页面
- 选课页面
- 成绩查询页面
- 课程资源页面
- 作业管理页面
- 通知公告页面
- 新闻浏览页面

**后台管理页面：**
- 管理员控制台
- 用户管理页面
- 课程管理页面
- 成绩管理页面
- 信息发布管理页面
- 系统设置页面

### 6.3 交互要求
- 表单验证提示清晰
- 操作反馈及时（Loading状态、成功/失败提示）

- 支持键盘快捷操作
- 分页加载，提升性能

## 7. 技术实现要点

### 7.1 前端技术栈
- **框架**: Vue 3 + Vue Router + Pinia（状态管理）
- **UI组件库**: Element Plus 或 Ant Design Vue
- **HTTP客户端**: Axios
- **构建工具**: Vite
- **其他**: ECharts（数据可视化）、Day.js（时间处理）

### 7.2 后端技术栈
- **框架**: Spring Boot 2.7+
- **ORM**: MyBatis Plus
- **安全**: Spring Security + JWT
- **数据库**: MySQL 8.0
- **缓存**: Redis
- **文件存储**: 本地存储或阿里云OSS
- **其他**: Lombok、Hutool工具类

### 7.3 关键技术实现
- **权限控制**: 基于RBAC模型，使用拦截器和注解实现
- **文件上传**: 支持多文件上传，限制文件类型和大小
- **数据导入导出**: 使用EasyExcel处理Excel文件
- **定时任务**: 使用Spring Task处理定时提醒
- **日志管理**: 使用AOP记录操作日志

## 8. 项目实施计划

### 8.1 开发阶段划分

**第一阶段：需求分析与设计（1-2周）**
- 完成需求文档编写
- 设计数据库表结构
- 设计系统架构和接口文档
- 设计UI原型图

**第二阶段：环境搭建与基础开发（1周）**
- 搭建前后端开发环境
- 创建项目基础框架
- 配置数据库连接
- 实现用户登录认证模块

**第三阶段：核心功能开发（4-5周）**
- 用户管理模块（1周）
- 教务管理模块（2周）
  - 课程管理
  - 选课系统
  - 成绩管理
  - 课表管理
- 信息发布模块（1周）
- 在线学习模块（1周）

**第四阶段：测试与优化（1-2周）**
- 功能测试
- 性能优化
- Bug修复
- 用户体验优化

**第五阶段：部署与文档编写（1周）**
- 系统部署上线
- 编写用户手册
- 编写毕业设计论文

### 8.2 开发优先级
**高优先级（核心功能）：**
- 用户登录认证
- 课程管理和选课系统
- 成绩查询
- 课表展示

**中优先级：**
- 通知公告发布
- 作业管理
- 课程资源管理

**低优先级（可选功能）：**
- 数据统计分析
- 高级搜索功能
- 消息推送

## 9. 风险评估与应对

### 9.1 技术风险
- **风险**: 对Vue3或Spring Boot不够熟悉
- **应对**: 提前学习相关技术文档，参考开源项目

### 9.2 进度风险
- **风险**: 功能开发超出预期时间
- **应对**: 优先完成核心功能，次要功能可适当简化

### 9.3 性能风险
- **风险**: 并发访问时系统响应慢
- **应对**: 使用Redis缓存，优化数据库查询，添加索引
