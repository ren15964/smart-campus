# 管理系统（通用后台）需求示例（复制给AI即可从零搭建）

## A. 项目一句话说明

- **项目名称**：企业内部管理系统（通用后台）
- **题材类型**：管理后台
- **核心目标**：为管理员提供“组织/员工/公告/工单/资产”一体化管理，支持权限控制与审计，方便演示与答辩。

## B. 默认技术选型

- **后端**：Spring Boot 2.7.x + Maven + MyBatis Plus + JWT
- **前端**：Vue 3 + Vite + Element Plus + Pinia + Axios + Vue Router
- **数据库**：MySQL 8.x
- **缓存/消息**：Redis（可选；若不装则代码需可降级运行）
- **接口风格**：RESTful
- **部署方式**：本地运行即可

## C. 角色与权限

- **角色列表**：
  - 管理员：admin：员工管理、角色权限、公告发布、工单处理、资产管理、系统看板
  - 普通员工: 员工信息查看/修改（仅自己）、提交工单、查看公告
- **权限粒度**：菜单+按钮级（加分；若实现成本高可先菜单级，但接口仍需角色校验）

## D. 功能模块清单

### D1. 必备基础模块

- 认证模块：登录/退出/获取当前用户（JWT）
- 用户中心：个人信息、修改密码
- 后台管理框架：Layout（侧边栏/顶部栏/面包屑）

### D2. 业务模块

1) 员工管理（employee）
   - 页面：员工列表（搜索/分页）、新增/编辑员工、员工详情
   - 规则：
     - 管理员可管理全部员工
     - 普通员工只能查看/编辑自己的基础资料（如头像/电话/邮箱）
2) 公告管理（notice）
   - 页面：公告列表、公告详情、公告发布（管理员）、已读标记（员工）
   - 规则：公告支持置顶、支持按标题关键字搜索
3) 工单管理（ticket）
   - 页面：工单列表、提交工单（员工）、处理/指派/关闭（管理员）
   - 规则：工单状态流转：`新建 -> 处理中 -> 已解决/已关闭`（不允许倒退）
4) 资产管理（asset）
   - 页面：资产列表、资产入库/编辑、资产分配/归还
   - 规则：资产状态：在库/已分配/维修中/报废

### D3. 加分项（可选）

- 首页数据看板（ECharts）：员工数、未处理工单数、资产状态分布、近7日新增工单趋势
- Excel 导出：员工列表导出

## E. 数据模型

### E1. 实体清单

- User（用户）
  - 字段：username, password(BCrypt), role_code, nickname, phone, email, avatar_url, status
- Employee（员工档案）
  - 字段：user_id, emp_no(唯一), dept, position, join_date, status
- Role（角色）
  - 字段：code(唯一), name
- Permission（权限）
  - 字段：code(唯一), name, type(menu/button), parent_code, path, component, icon, sort
- UserRole（用户-角色）
  - 字段：user_id, role_code
- Notice（公告）
  - 字段：title, content, is_top, publish_time, publisher_id
- NoticeRead（公告已读）
  - 字段：notice_id, user_id, read_time
- Ticket（工单）
  - 字段：title, content, status, creator_id, handler_id, priority, close_time
- Asset（资产）
  - 字段：asset_no(唯一), name, category, status, buy_date, price, assigned_to_user_id

### E2. 关系

- User 1-1 Employee
- User N-M Role（UserRole）
- Role 1-N Permission（或 Role-Permission 中间表，若要更规范可加 RolePermission）
- Notice 1-N NoticeRead
- User 1-N Ticket（creator）
- User 1-N Asset（assigned_to）

## F. UI/交互要求

- **整体风格**：清爽现代管理后台（可选“精致工业风”）
- **管理页形态**：搜索区 + 表格 + 分页 + 新增/编辑弹窗 + 删除确认 + loading + 空态
- **提示与异常**：成功/失败 toast；401 跳登录；403 无权限页

## G. 接口与错误码约定

- Header：`Authorization: Bearer <token>`
- 统一返回：`{ code, msg, data }`
- 分页参数：`pageNum`(从1) `pageSize` `sortBy` `sortOrder`
- 语义：401 未登录，403 无权限，400 参数错误，500 服务器异常

## H. 交付物清单（必须全部产出）

1) `database.sql`（含初始化：admin/admin123、employee/123456）
2) 后端工程（分层、JWT、异常、统一返回、分页模板、至少 2 个业务模块完整 CRUD）
3) 前端工程（登录+Layout+至少 2 个管理模块页面 CRUD）
4) 文档：README（启动+账号）、API接口文档（全量接口+示例）
5) 验收：登录→首页看板→员工 CRUD→工单状态流转
