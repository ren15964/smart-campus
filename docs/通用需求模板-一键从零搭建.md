---

## G. 接口与错误码约定（建议照抄）

- **请求头**：`Authorization: Bearer <token>`
- **统一返回**：
  - `code`：200 成功，其它为错误码
  - `msg`：提示信息（中文）
  - `data`：返回数据
- **分页参数**：`pageNum`(从1) `pageSize` `sortBy` `sortOrder`
- **HTTP 状态语义**：401 未登录，403 无权限，400 参数错误，500 服务器异常

---

## H. 交付物清单（AI 必须全部产出）

1) **数据库**：`database.sql`（建表 + 索引 + 初始化账号数据）
2) **后端工程**：
   - 分层结构（controller/service/mapper/entity/dto/vo）
   - JWT 登录鉴权、全局异常、统一返回、分页查询模板
   - 至少 1 个业务模块完整 CRUD（按你 D/E 的模块）
3) **前端工程**：
   - 登录页 + Layout（菜单/路由）+ 至少 1 个业务模块管理页（CRUD）
   - axios 封装（token 注入、401 处理）、路由守卫、Pinia 持久化
4) **文档**：
   - `README.md`：环境要求、启动步骤、默认账号
   - `API接口文档.md`：所有接口清单 + 示例
5) **验收方式**（AI 需在 README 写明）：
   - 后端启动成功
   - 前端启动成功
   - 登录→进入首页→访问受保护接口→完成一次 CRUD 演示

## I. AI 执行指令（强制，建议原样保留）

请你严格按以下流程自动实现，不要只给建议：

1. 先输出“你将生成的目录树/文件清单”（简洁即可）。
2. 生成 `database.sql`（含初始化账号：管理员/普通用户）。
3. 生成后端工程：可启动、可连接数据库、具备 JWT 鉴权与示例模块。
4. 生成前端工程：可启动、可登录、可进入管理页并调用后端接口。
5. 同步生成/更新 `README.md` 与 `API接口文档.md`。
6. 最后给出 Windows 下的启动命令（PowerShell 版本）。
